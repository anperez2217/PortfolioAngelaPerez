Key Page Breakdowns

FAQ Page

Built as a Blazor component that displays a list of frequently asked questions from a JSON file. It subscribes to an event service, so when an admin updates the FAQs, the user page refreshes automatically in real time using an accordion UI for easy reading.

Users Feedback Page

Implemented as a Blazor form component where users can submit feedback. Input is validated on the client side before being saved to a JSON file via a dedicated service. After submission, users see a confirmation message.

Admin FAQ Page

A Blazor Server page for admins to perform CRUD operations on FAQ items, with changes saved asynchronously. Event-driven updates refresh user FAQ pages instantly. It also displays user feedback for admin review, using dependency-injected services for security.

Thank You Page

Displays after checkout to confirm the order with a logo and thank-you message, providing visual confirmation that the transaction is complete. Order details are stored in the app state, and navigation back to the cart is disabled.

How It Was Built

Backend Architecture

  • C# ASP.NET Core 8.0
    Built request handlers with async/await, implemented service layer with DI.
  • Entity Framework Core ORM

  • SQLite Database
    Designed schema for users, carts, orders, FAQs, implemented cookie sessions.
  • HtmlSanitizer Security
    Sanitized admin FAQ input, implemented CSRF tokens on forms.
